# 두 번째 회고

이번주는 수업을 들으며 웹 접근성에 대해 좀 더 알게되는 한 주 였습니다. 그 중 하루는 수업만 듣고 쉬는 책임없는 쾌락을 즐겼습니다.
wai-aria를 배울 때 잠이 쏟아졌습니다. 운동을 함으로 인해 수업시간에 허리통증은 해결되었지만 잠이오는 부작용이 생긴것 같습니다.
아직 운동도 수업도 초반이라 그 적정선을 찾는것도 하나의 숙제인것 같습니다.
그래도 다행인 것은 선생님께서 알려주신 정보로 미리 예습을 했습니다.
실질적으로 어떻게 사용되는지 그리고 제대로 이해가 안된 부분은 수업시간을 통해 해결할 수 있었습니다.
아직 사용하기에 미숙하지만 그래도 조금이나마 발전을 할 수 있었습니다.
그리고 디자인에 따라 마크업을 작성한다기보다 목적에 맞는 태그를 사용하는 것도 주의를 해야겠습니다.

extention이라던지 환경설정 부분에서 선생님의 노하우가 들어가있기 때문에 올리는게 맞지 않다고 생각했습니다.
만약 다시 작성하게 된다면 어떻게 작성해야할지도 찾고 고민을 해봐야겠습니다.

# 웹개발 챌린지

우선 프론트엔드 개발자는 퍼블리셔와 백엔드 사이에서 다리같은 존재라고 생각합니다. 그렇기때문에 백엔드 실무자만큼은 아니지만 어느정도의 지식은 필요하다고 생각했습니다.
<br />
멋사의 vod입니다. 그 중 todo프로젝트는 진행하지 않았고
spring boot는 어떻게 사용해야되는지
mariaDB를 사용했으며
jsp를 사용하는 간단한 방법정도 따라해보았습니다.

## vod를 선택한 이유는

1. spring boot : 현업에서 많이 사용하기도 하고 안정성이나 성능이 더 좋다고 보았습니다. -필요하다면 좀 더 알아봐야될 필요가 있을것 같습니다
2. mongoDB를 사용했다면 noSql이기때문에 더 쉽게 사용 가능했겠지만 db를 연동해서 sql문을 사용해보는것도 중요하다고 생각했습니다
3. jsp의 사용 : 수업때는 html, css, javaScript를 배우고 react에서는 jsx를 사용하기 때문에 좀 더 다양하게 해봐야겠다 생각을 했습니다.
4. xml 사용: 마크업 언어로 이루어 진건 이해했는데 어떻게 사용하는지 잘 몰랐습니다. 이 vod를 통해 mapper를 사용해서 서로 연결하는 것을 보면서 이해를 할 수 있었습니다.
